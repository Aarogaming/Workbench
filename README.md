# Workbench

## Overview
Workbench is a collection of cross-project scripts, utilities, and automation tools used throughout the workspace. It provides shared functionality for development, testing, and deployment across multiple projects.

## Features
- PowerShell and Python scripts for automation
- Asset management and ingestion tools
- Environment and dependency checkers
- Utilities for CI/CD and local development

## Getting Started

### Prerequisites
- Python 3.12+ (for Python scripts)
- PowerShell 7+ (for PowerShell scripts)
- (Optional) Node.js for JS-based tools

### Setup
1. Clone the repository and navigate to the Workbench directory.
2. Review available scripts in the root and `Assets/` directories.
3. Install any required dependencies as noted in script headers or `py-env.md`/`dotnet-env.md`.

## Usage
- Run scripts directly from the command line or integrate them into CI/CD pipelines.
- See the `README.md` or script headers for usage instructions.
- Contribute new tools by following the guidelines below.
